STEP 1
발견
내 업무 중 번거롭게 느껴지는 것을 찾아보세요
TIP
"또 이거 해야 하네..." 하고 떠오르는 업무가 있나요?
매번 비슷한 패턴으로 반복하는 업무를 떠올려보세요. ▼ 더 자세한 가이드 보기
📌 이런 업무를 떠올려보세요
  • 매주/매월 반복하는 보고서 작성
  • 같은 형식의 데이터를 정리하거나 옮기는 작업
  • 매번 비슷한 이메일, 공문, 안내문 작성
  • 여러 파일/시스템에서 데이터를 모아 합치는 일
💡 작성 요령
  • "번거로운 업무"는 한 문장으로 핵심만 적으세요
  • "빈도/시간"은 대략적으로 OK (예: 매주 1회, 약 2시간)
  • "존재 이유"는 왜 이 업무를 하는지 — 보고 의무, 법적 요건 등
번거롭게 느껴지는 업무는 무엇인가요?
얼마나 자주, 얼마나 오래 걸리나요?
그럼에도 이 업무가 존재하는 이유는 뭘까요?
STEP 2
정의
이 업무의 시작(Input)과 끝(Output)을 명확하게
TIP
"이 업무를 시작할 때 내 손에 뭐가 들려있지?" → Input
"이 업무가 끝나면 뭐가 완성되어 있어야 하지?" → Output ▼ 더 자세한 가이드 보기
📌 Input/Output 예시
  • 보고서 작성 → Input: 원시 데이터 파일 / Output: 완성된 보고서
  • 이메일 답장 → Input: 고객 문의 내용 / Output: 답변 이메일
  • 데이터 정리 → Input: 여러 부서 엑셀 파일 / Output: 통합 정리표
💡 작성 요령
  • Input: 이 업무를 시작할 때 내가 받는 것 (파일, 데이터, 요청 등)
  • Output: 이 업무가 끝났을 때 만들어지는 것 (문서, 보고서, 메일 등)
  • 최대한 구체적으로 적을수록 좋아요
INPUT — 무엇을 넣으면
AI
OUTPUT — 무엇이 나오길 바라는지
STEP 3
분해
Input과 Output 사이에 어떤 단계들이 있나요?
TIP
앞서 정의한 Input(시작)에서 Output(끝)까지, 실제로 하는 순서를 적어보세요.
더 잘게 쪼갤수록 좋아요.
"시간이 오래 걸린다", "실수가 난다", "귀찮다"는 단계에 비효율 버튼을 눌러주세요. ▼ 더 자세한 가이드 보기
📌 분해 예시
  1. 각 부서에 데이터 요청 메일 발송
  2. 회신 데이터를 엑셀에 취합
  3. 수치 검증 및 이상값 확인
  4. 보고서 양식에 데이터 입력
  5. 상사 검토용 초안 작성
💡 작성 요령
  • 실제로 하는 순서대로 적으세요
  • 3~7단계가 적당해요
  • "비효율" 버튼: 시간이 오래 걸리거나 실수가 잦은 단계에 표시
업무를 단계별로 쪼개보세요
STEP 4
전환
작성한 내용을 AI에게 전달할 프롬프트로 전환합니다
TIP
1~3단계에서 정리한 내용이 자동으로 프롬프트에 반영됩니다.
이 프롬프트를 AI에게 보내면, AI가 당신의 상황을 파악하기 위한 질문을 던져줍니다. ▼ 더 자세한 가이드 보기
📌 프롬프트 사용법
  1. "복사" 버튼을 눌러 프롬프트를 복사하세요
  2. ChatGPT, Claude 등 AI 도구에 붙여넣으세요
  3. AI가 상황을 파악하기 위한 질문을 던져줍니다
  4. 질문에 답하면서 AI와 함께 해결 방법을 찾아보세요
💡 팁
  • 프롬프트를 수정하고 싶으면 "수정하기"를 눌러 이전 단계로 이동
  • "다운로드"로 텍스트 파일 저장, "이메일로 받기"로 메일 수신 가능
